# 话饼 Web Vue2 门店订货流

## 开始前

1. 读取 `references/shop-indent.md`。
2. 检查宿主项目的 API、状态层、页面、路由和枚举来源，确认哪些层已经存在。
3. 明确操作属于创建、编辑、复制式创建、详情状态操作还是导出，并确认服务端是否负责价格汇总。

## 实施流程

1. 列表页持有筛选和分页；按共享范围决定直接请求 API 还是通过 Vuex action 获取记录、总数与 loading。
2. 创建/编辑先获取基础回显与可选分类商品，按稳定商品 ID 合并数量，并生成页面本地可编辑副本。
3. 商品数量变化时构造最小参数请求服务端汇总；使用防抖与单调 revision/latest-wins，只接纳最新输入对应结果。
4. 提交前 flush 并等待最新汇总，从分类中去重提取 `quantity > 0` 的商品，再构建创建或编辑 DTO。
5. 详情页按状态显示编辑、取消、完成、备注和导出等宿主允许的入口；操作成功后重新获取权威详情。
6. 新增路由时遵循宿主静态、约定式或动态菜单策略，不复制固定 path。

## 约束

- 是否使用 namespaced Vuex、是否持久化和 action 命名均以宿主项目为准；页面私有数据不必强行进入 store。
- 分类数量初始化必须写入真实商品项，不能误写到分类对象。
- 子表格通过 emit 上报 `{ goodsId, quantity }` 等最小变更，不用嵌套 `v-model` 修改 prop 或 getter。
- 金额 `0`、数量 `0` 和空列表分别处理；金额使用服务端权威汇总或精确小数策略。
- 筛选改变时重置页码，分页 size/current 各只触发一次查询。
- 状态值与文案从宿主 options/类型层解析，不在组件中自造枚举。
- 不重复包装 Promise，不保留调试日志或空 catch。

## 验证

1. 覆盖列表搜索、重置、分页、空列表和失败后的 loading。
2. 覆盖创建/编辑回显、分类切换、数量 0/正数、汇总乱序、重复提交和返回列表。
3. 覆盖详情状态、取消、完成、备注、导出文件名与刷新。
4. 核对页面 → API/Vuex → 汇总 → DTO → 详情及路由参数的完整链路。
5. 运行宿主项目定向 lint/测试并审查 diff；构建前确认无部署、推送、通知或发布副作用。
