# 话饼 Vue3 小程序首页看板

## 先识别首页模型

不要假设首页路径、store 名、endpoint 或权限码。先完成以下工作：

1. 从 `pages.json` 与导航配置确认真实首页和可能的多角色首页。
2. 读取首页页面、同级组件、状态模块、API 模块、权限工具和跳转工具。
3. 标出每个卡片或入口的“接口 → 状态 → 展示 → 动作”链路。
4. 区分首屏必需数据、可延迟数据、静态配置、菜单权限和敏感指标权限。
5. 确认返回首页、下拉刷新、热启动、账号/租户切换时的刷新策略。

需要为陌生项目建立完整盘点时，读取 `references/adaptation-checklist.md`。

## 保持清晰的数据分层

优先使用以下职责划分，并服从目标项目已有约定：

1. API 层声明 endpoint 和请求/响应类型，不持有 UI 状态。
2. store/composable 保存领域数据、执行请求和数据整形，不直接操作页面 toast 或跳转。
3. 首页页面编排生命周期、请求分组、刷新和页面级状态。
4. 卡片与快捷入口组件负责展示、局部交互和明确事件，不各自复制请求逻辑。
5. 只有跨页面复用的权限、格式化和跳转能力才进入公共工具。

新 action 可直接使用 `async/await`，不要无额外语义地再包一层 `new Promise`。给指标、菜单项和权限集合补最小必要 interface，避免继续扩散 `any`。

## 设计刷新与缓存策略

- `onLoad` 适合只依赖初始参数的一次性初始化；`onShow` 适合从其他页面返回后必须刷新的数据。
- 不要机械地把全部请求都放进 `onShow`。为每类数据定义新鲜度、触发条件和去重策略，避免频繁切页造成请求风暴。
- 账号、租户、门店或角色变化时清空关联统计、菜单和权限；未知身份阶段按无权限处理。
- 持久化首页数据时定义版本、TTL 和身份维度。实时指标通常不应无期限持久化。
- 下拉刷新与自动刷新共用同一 refresh action，并保证并发锁、取消和 loading 状态不会互相覆盖。

## 编排首屏请求

- 把首屏必需请求、可延迟请求和独立卡片请求分组，不让低优先级数据阻塞主内容。
- 全有或全无的依赖使用 `Promise.all`；允许部分成功的卡片使用 `Promise.allSettled` 或独立状态。
- 用 `try/catch/finally` 闭合 loading；不要只在成功分支设置 `requestOver`。
- 为 page、section 或 card 选择合适粒度的 loading/empty/error，避免一个接口失败遮住整个首页。
- 错误文案使用安全兜底，不直接解引用不稳定的 `err.data.msg`。
- 零值是合法指标，不能用普通 truthy 判断把 `0` 当成空数据。

## 分离权限层级

至少区分三类权限：

- 页面/菜单权限：入口是否可进入；
- 动作权限：按钮是否可点击或显示；
- 数据权限：敏感指标是否允许渲染。

处理时遵守以下规则：

- 从项目真实权限源读取，权限 code、菜单根节点和路由 path 都通过配置或类型表达，不写死示例值。
- 权限集合先归一化为空数组，再使用 `some/includes/find`；不要在用户信息未就绪时直接 `.filter()`。
- 权限未知阶段不渲染敏感值，防止“先显示后隐藏”的数据闪现。
- 无页面权限时阻止跳转并使用项目统一反馈；无数据权限时隐藏或脱敏，不把接口值传给无权组件。
- 后端仍是最终授权边界，前端权限只能控制体验，不能替代接口鉴权。

## 构建卡片与入口组件

- 使用项目现有 UI 库、主题 token 和组件约定，不在同一首页混入另一套组件库。
- 小程序公共模板优先使用 `view`、`text`、`image`；H5 标签仅在已有兼容策略或条件分支中使用。
- 继续使用项目约定的 rpx/Tailwind/SCSS 体系，验证图片 mode、数字动画、长文本、安全区和不同屏宽。
- 数字动画要处理初始值、负数、小数精度、接口刷新和 reduced motion；无数据时不要从旧值动画到错误值。
- 快捷入口先做权限检查，再选择原生导航、tabBar、分包页或 WebView 跳转。
- WebView token、query 编码和域名 allowlist 归 `huabing-mini-vue3-runtime`，首页只维护业务入口。

## 避免把单项目细节固化为规范

- 不复用固定 endpoint、store id、菜单根 path、权限 code、H5 页面路径或图片资源名。
- 不把提交历史中的临时日志、mock 数据、未消费状态和注释代码写进新实现。
- 不恢复已删除页面或修改 vendored UI 源码，除非当前需求明确要求。
- 先匹配目标项目的 Composition API/Options API、Pinia/Vuex 和 UI 库；Vue 3 不等于所有项目都使用同一写法。

## 文档与交付

- 面向项目所有者的 constitution、spec、plan、tasks、checklist、README、手册、prompt 模板说明、metadata 说明和人工评审记录使用中文主文。
- 命令、路径、文件名、代码标识符、JSON key、API 字段、模型名和第三方专有术语保留英文，避免为翻译牺牲准确性。
- 引用英文资料时给出中文结论摘要；只有英文主文能显著降低工具误解时才使用英文，并同时补充中文解释。

## 边界

- 页面注册、路由壳层、插件和多端构建归 `huabing-mini-vue3-app-shell`。
- token、请求 client、持久化和 WebView 安全归 `huabing-mini-vue3-runtime`。
- Vue 2 uni-app 项目应使用独立的 `huabing-mini-vue2-*` 系列，不要直接复制 Vue 3 Composition API 或 Pinia 实现。
- 大规模首页改版先建立 spec/plan；本技能用于实现已明确的看板结构和数据契约。

## 验证

- 运行目标仓库 lint、typecheck、单测和主要小程序端构建。
- 手测首次进入、返回首页、下拉刷新、快速重复切换、部分接口失败、全空、零值和慢网。
- 覆盖账号、租户、门店与角色切换，以及无菜单权限、无动作权限、无数据权限。
- 检查页面日志、快照和错误上报不包含敏感指标、token、完整认证 URL 或真实用户数据。
