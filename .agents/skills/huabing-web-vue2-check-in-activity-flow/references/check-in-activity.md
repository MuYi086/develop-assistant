# 打卡与任务活动通用业务地图

## 实体层级

```text
活动 Activity
  -> 任务 Task
       -> 完成条件 Condition
       -> 奖励 Reward
  -> 活动状态与适用范围
```

先把宿主接口字段映射到这些角色。活动 ID、任务 ID、来源活动/任务 ID 和目标活动/任务 ID 必须单独命名，不能依赖对象展开碰巧携带。

## 能力分组

| 层级 | 常见动作 |
| --- | --- |
| 活动 | 列表、详情、创建、更新、上线、关闭 |
| 任务 | 列表/路线、创建、更新、排序、删除 |
| 完成条件 | 保存、详情、覆盖、复制、应用全部任务 |
| 奖励 | 配置、回显、更新、库存或预算校验 |

- API 函数名遵循宿主项目规则，但注释必须描述真实业务动作。
- “来源活动全部任务条件 → 目标活动全部任务”是活动级批量操作，不应在浏览器中循环调用任务保存接口造成部分成功。
- 若后端没有原子接口，先明确任务匹配键、幂等策略、失败补偿和进度反馈，再决定是否实施。

## ID 与 DTO

| 上下文 | 用途 | 常见来源 |
| --- | --- | --- |
| 活动 ID | 详情、编辑、上线、关闭 | 路由参数或列表行 |
| 任务 ID | 编辑、删除、奖励、条件 | 任务路线行 |
| 来源 ID | 复制条件或模板 | 复制弹窗选择 |
| 目标 ID | 接收复制或覆盖 | 当前详情上下文 |

- 接口可能用 number、string 或兼容字段；在 API 边界显式转换，不全局替换字段名。
- 仅从白名单构建 DTO，避免把列表状态、显示字段和来源审计字段带入提交。
- 上抛行操作时使用副本，例如 `{ ...row, contextId }`，不得修改共享行对象。

## 弹窗生命周期

```text
父级 open(mode, context)
  -> 子组件构建回显副本并 reset validation
  -> 用户编辑与校验
  -> 子组件 emit 提交意图/DTO
  -> 父级或 store 调用 API
  -> 成功：close + 刷新详情
  -> 失败：保留输入 + 恢复 loading
```

- 公共 Dialog 的确认、取消、辅助按钮和关闭图标是不同事件。
- 提交锁只作用于确认路径，取消不应被失败的提交永久禁用。
- 富文本、上传和异步选项必须支持编辑回显；销毁或切换实体后旧请求不得写回。

## 状态机

实际状态码由宿主接口决定，至少建模：

| 状态角色 | 典型权限 |
| --- | --- |
| 草稿 | 可编辑活动、任务、条件和奖励 |
| 待开始/已上线 | 限制结构性修改，可查看配置 |
| 运行中 | 只允许产品明确的动态调整 |
| 已结束/已关闭 | 只读，保留审计和结果入口 |

前端按钮权限用于减少误操作，服务端仍必须校验状态迁移。上线、关闭和删除需要二次确认，并在成功后重新获取状态。

## 回归矩阵

- 活动级：创建、编辑、上线、关闭、结束显示。
- 任务级：增删改、排序、完成条件与奖励回显。
- 复制：单任务、全部任务、来源为空、目标不匹配、重复提交和后端失败。
- 弹窗：重复打开、取消、确认失败、异步选项晚到和切换目标。
- 外链：按跳转类型验证 URL、参数、空值与安全协议白名单。
