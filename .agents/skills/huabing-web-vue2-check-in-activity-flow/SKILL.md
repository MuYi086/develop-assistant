# 话饼 Web Vue2 打卡活动流

## 开始前

1. 读取 `references/check-in-activity.md`。
2. 检查宿主项目的活动、任务、条件与奖励 API，列出状态机、ID 字段和弹窗契约。
3. 确认需求作用于活动级、任务级、完成条件级还是奖励级，避免混用上下文。

## 实施流程

1. 列表页负责查询、筛选、状态入口和跳转；详情页负责任务路线及各弹窗的流程编排。
2. 弹窗维护表单副本、校验与回显，通过明确的 open 参数初始化，通过事件把提交意图交给数据拥有者。
3. 按接口契约显式映射活动 ID、任务 ID、来源 ID 和目标 ID，不通过整行对象残留隐式带入。
4. 创建、编辑、复制条件、覆盖条件和应用全部任务分别建模；批量复制必须由后端原子接口或可恢复的事务流程承载。
5. 成功后关闭弹窗并刷新权威详情；失败时保留用户输入并恢复 loading。
6. 按活动和任务状态控制按钮可见性、可编辑性与二次确认，不能只依赖后端报错。

## 约束

- 同页并发请求使用独立 loading 或请求标识，保证每个异常路径都收口。
- 行事件上抛时先复制，再追加活动上下文，避免污染列表行或 Vuex 状态。
- 富文本与上传组件支持异步回显，但 watcher 不得形成双向回写循环。
- URL、跳转类型、奖励和完成条件都以宿主产品规则与接口枚举为准。
- 取消、关闭图标、确认与双按钮操作必须有独立事件语义。
- 不保留调试日志、空 catch、错误 API 注释或来源项目的专属字段假设。

## 验证

1. 覆盖列表查询、创建、编辑、关闭/上线、进入详情和返回刷新。
2. 覆盖任务增删改、奖励、条件保存、全部应用、跨活动复制和部分失败防护。
3. 重复打开每个弹窗，确认表单、校验、选择项和 loading 不串数据。
4. 核对草稿、运行中、结束和关闭状态的权限、显示与接口动作。
5. 运行宿主项目定向 lint/测试并审查 diff；构建前确认不会触发部署、推送、通知或发布。
