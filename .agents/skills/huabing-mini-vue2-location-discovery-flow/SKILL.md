# UniApp Vue 2 定位与内容发现

## 状态模型

显式区分：

- 权限：`unknown`、`granted`、`denied`、`system-disabled`。
- 来源：GPS、用户选点、缓存、城市默认中心点。
- 坐标：原始坐标系、转换后坐标、精度和获取时间。
- 发现结果：loading、success、empty、error、stale。

不要把“使用默认中心点”伪装成“已授权定位”。

## 初始化顺序

1. 先处理目标平台隐私要求，再请求位置权限。
2. 通过项目统一 location adapter 调用 `uni.getLocation`、选择位置或设置页，不在多个页面复制权限逻辑。
3. 归一化经纬度、坐标系、城市信息和错误原因后写入 location Store。
4. 首屏核心内容与附近推荐按产品依赖决定并发或分离；较慢推荐不应无条件阻塞首屏。
5. 每条独立 Promise 都处理失败和收尾，不能因为未进入 `Promise.all` 而产生未处理拒绝。

## 坐标与查询

- 明确后端、地图 SDK 和 `uni.getLocation` 使用的坐标系；转换只在 adapter 边界进行一次。
- 请求携带位置来源和精度时遵守后端契约，不伪造高精度。
- 登录专属的收藏/常用 tab 通过统一登录流程，附近列表可按后端能力提供匿名 endpoint。
- 搜索词、位置、筛选和分页共同组成 query key，切换任一条件时重置分页并防止旧响应覆盖。

## 缓存与隐私

- 缓存最小位置摘要和少量推荐项，设置过期时间；不持久化完整轨迹。
- 列表过大时优先不持久化或截断，不依赖移动端 storage 保存全部结果。
- 用户撤回权限或注销时清理精确位置和用户关联缓存。
- 上传日志时降低坐标精度或脱敏，遵守项目隐私策略。

## 降级

- 拒绝授权：展示解释、手动选点或城市选择入口。
- 系统定位失败：使用最近有效位置或配置的默认中心点，并标记来源。
- 附近为空：区分无服务点与定位失败，不共用同一个空态。
- 地图/平台能力不可用：提供列表或地址文本降级。

## 验证

- 权限未知、同意、拒绝、系统关闭、设置页返回和权限撤回。
- GPS、用户选点、缓存和默认中心点四类来源。
- 坐标转换、附近/收藏/搜索、分页、快速切换查询和空数据。
- 微信、支付宝、H5/App 的权限声明、API 返回和地图能力差异。
