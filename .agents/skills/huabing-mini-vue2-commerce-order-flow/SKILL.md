# UniApp Vue 2 交易与订单流

## 建立闭环

按实际产品画出：

```text
商品选择 -> 购物车 -> 结算草稿 -> 服务端试算 -> 创建订单
       -> 支付 -> 订单详情 -> 履约/取货/配送 -> 完成或售后
```

为每个节点标明输入、输出、状态所有者、幂等键、可返回页面和清理点。不要只修当前页面而忽略贯穿流程的标识符。

## 购物车

- 匿名与登录购物车只有在后端模型确实区分时分别维护；登录后迁移必须幂等且可重试。
- cart id/key、门店/仓、配送或用餐方式等上下文在加减购、提交和结算间保持一致。
- 加购、减购、删除、清空和提交保持明确 command，不用一个巨型函数靠参数猜动作。
- 乐观更新必须有回滚；库存、限购和价格仍以服务端结果为准。
- 重复点击通过表单提交技能的 single-flight 锁处理。

## 结算与试算

- 区分 committed 选择与 popup draft：优惠、配送、地址、支付方式等取消后不能污染正式状态。
- 任一影响价格的字段变化后，以稳定依赖顺序重新试算；防止 watcher 互相触发形成请求循环。
- 金额、优惠、库存、运费和可用渠道以最新服务端试算为权威，页面只派生展示。
- 创建订单前重新验证关键条件，并携带服务端幂等键或客户端 request id。
- 页面返回刷新使用明确 dirty flag，不在每次 `onShow` 无条件重放全部请求。

## 标识符与状态

- 在入口一次性归一 `orderId`、transaction id、parent/child id、联合单标志和布尔/数字 query。
- 内部使用一种规范字段，旧字段只在 adapter/route 边界转换。
- 订单与履约使用显式状态机；未知状态保留原值并进入安全降级，不把未知枚举当成功。
- 展示 label/style 从状态派生，服务端枚举不被 UI 文案覆盖。

## 路由与实时能力

- 多版本详情、取货类型和跳转语义使用 `huabing-mini-vue2-versioned-route-flow` 的表驱动矩阵。
- 支付使用 payment-channel 技能，取消/退款使用 order-after-sales 技能。
- WebSocket、轮询、二维码过期和页面 timer 使用 realtime-lifecycle 技能，订单页只消费领域事件。
- 全局实时组件通过事件或 Store 发布领域消息，不直接操作具体页面实例。

## 验证

- 匿名/登录、购物车迁移、重复提交、库存变化和价格变化。
- 优惠/配送/地址/支付方式的选择、取消、回显和返回刷新。
- 创建订单成功、业务失败、网络超时、重试与幂等。
- 单/联合单、不同履约类型、旧/新详情、实时更新和页面卸载。
