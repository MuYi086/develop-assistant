# Vue 2 页面与组件通用模式

## 宿主项目适配清单

开始编码前先确认：

- Vue、Vue Router、Vuex 与 UI 组件库版本；
- `views`、`components`、`api`、`store`、`router` 和样式目录的实际位置；
- 静态路由、文件约定路由或后端动态菜单的注册方式；
- 请求封装、权限指令、全局组件、路径别名和 lint 规则；
- 相邻页面的命名、表单校验、弹窗与 loading 契约。

不得把某个来源项目的路径、作者习惯或组件名当成通用前提。当前宿主代码与接口契约始终优先。

## 组件责任

| 角色 | 主要责任 | 不应承担 |
| --- | --- | --- |
| 页面容器 | 路由参数、权限、查询、流程编排、刷新 | 复杂控件内部细节 |
| 展示组件 | 表格、卡片、详情块及用户事件 | 跨页面跳转和全局状态决策 |
| 业务弹窗 | 表单副本、校验、回显、确认/取消 | 偷偷提交或修改父级对象 |
| 全局组件 | 跨业务稳定复用的交互或布局 | 特定业务字段和接口 |
| 通用工具 | 无界面的纯转换或校验 | UI 状态和副作用编排 |

## 数据与事件方向

```text
路由/页面容器 -> props -> 展示组件或弹窗
路由/页面容器 <- emit/确认结果 <- 展示组件或弹窗
页面容器 -> API 或 Vuex action -> mutation -> getter -> 页面
```

- 禁止用 `$parent` 穿透不稳定层级；使用 emit、显式回调或状态容器。
- 为创建、编辑、复制和查看建立显式 mode；每次打开弹窗都从输入重新构造表单副本。
- watcher 用于同步外部变化或触发受控副作用，不用于双向回写 prop。

## 路由选择

| 路由来源 | 实施要点 |
| --- | --- |
| 静态路由表 | 添加懒加载组件、权限 metadata 与明确的详情/编辑子路由 |
| 文件约定路由 | 遵循目录命名与生成规则，不重复手工注册 |
| 后端动态菜单 | 核对 path 到组件的解析器、菜单权限和非菜单子页白名单 |

无论采用哪种方式，都要全局搜索 path/name，并从列表、详情和外部入口验证参数一致性。

## 弹窗生命周期

```text
open(mode, context)
  -> 克隆回显数据并 reset validation
  -> 用户编辑与本地校验
  -> emit 提交意图和 DTO
  -> 数据拥有者请求 API
  -> 成功：关闭并刷新
  -> 失败：保留输入并恢复 loading
```

关闭图标、取消按钮和确认按钮是不同语义；只有确认路径需要提交锁。

## 样式与布局

- 优先使用宿主项目的设计 token、主题变量、间距工具和公共布局组件。
- 局部样式默认限制作用域；只有无业务语义且被多处复用的样式才进入全局层。
- 固定底部操作区要为长表单预留空间，并验证小屏、缩放和滚动容器。
- Element UI、Ant Design Vue 等组件库的尺寸与校验 API 按宿主项目适配，不混用不同版本写法。

## 质量底线

- `0`、`false` 与空字符串按字段语义分别处理，不用宽泛 truthy 判断。
- 异步状态使用 `try/finally` 或等价收口；旧响应不能覆盖新交互结果。
- 凭据、token、Webhook 和私有地址只能来自受控配置，不进入源码、日志或文档示例。
- 验证命令先检查 package scripts 的 pre/post 生命周期，避免把编译验证升级成发布动作。
