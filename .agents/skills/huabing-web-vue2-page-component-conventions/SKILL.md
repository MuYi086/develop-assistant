# 话饼 Web Vue2 页面组件规范

## 开始前

1. 读取 `references/page-component-patterns.md`。
2. 检查宿主项目的框架版本、目录结构、路由来源、UI 组件库、样式方案和相邻实现。
3. 把参考资料中的名称视为角色示例；不得直接复制目录、组件名或路由策略。

## 实施流程

1. 明确页面容器、展示块、业务弹窗、API/状态依赖和路由入口的责任边界。
2. 让页面容器编排查询、权限、路由和提交后刷新；把可独立理解的表格、卡片、详情块与复杂弹窗拆为局部组件。
3. 优先复用宿主项目现有的弹窗、上传、富文本、固定操作栏和布局组件；只有跨业务稳定复用的能力才提升为全局组件。
4. 遵循宿主项目的 Vue 2 Options API、命名、import、引号、分号和样式约定，不为统一格式改写无关代码。
5. 通过 props 下发只读数据，通过 emit 上抛业务意图；仅在组件明确暴露命令式接口时通过 `$refs` 调用 `show`、`hide` 或 `reset`。
6. 按宿主项目实际情况选择静态路由、文件约定路由或后端动态菜单；新增前搜索目标 path，避免重复注册。

## 约束

- 弹窗每次打开都显式初始化模式与回显副本，关闭时清理校验和临时状态；成功后通知拥有数据的上层刷新。
- loading 必须在成功、失败和取消路径收口；并发请求使用独立实例、计数器或请求标识，避免互相关闭。
- 不直接修改 prop、Vuex getter 返回值或复用中的列表行对象；先复制再编辑，提交时生成明确 DTO。
- 不复制空生命周期、调试日志、空 catch、重复 Promise 包装、硬编码凭据或无边界的大型组件。
- UI 组件库仅决定控件 API，不改变数据流、状态归属与验证原则。
- 面向项目所有者的说明、结论、注释与人工检查记录中文优先。

## 验证

1. 检查组件 import/注册、props/emit、`$refs`、路由唯一性和样式作用域。
2. 覆盖初次进入、重复打开、取消、提交成功、提交失败、快速重复点击和返回页面。
3. 优先运行宿主项目已有的定向 lint、单元测试或组件测试；构建前展开 npm/pnpm 生命周期并确认无部署、推送、通知或发布副作用。
4. 审查定向 diff 和敏感信息，确保没有无关重构。
