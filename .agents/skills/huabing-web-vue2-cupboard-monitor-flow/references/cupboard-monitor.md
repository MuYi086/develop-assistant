# 智能柜机与设备监控通用地图

## 数据分层

```text
筛选参数 -> 监控列表 API -> 页面记录
                      -> 同口径状态统计
视图偏好 -> 页面/可选 Vuex -> 卡片字段与顺序
设备 ID + 版本/能力 -> 详情 API -> 设备信息 + 库存 + 格口 + 传感器
日期 + 分页 -> 警报/操作日志 API -> 日志列表
```

- 视图偏好可共享或持久化；高频设备快照默认不持久化。
- 列表与统计必须使用同一筛选口径，或明确显示统计的独立口径。
- 写控制命令与只读监控是不同权限域，本 Skill 默认只处理只读监控。

## 多代设备适配

使用稳定的版本字段或 capabilities，不从某个可空子字段反推设备代际：

```text
raw response
  -> selectAdapter(version, capabilities)
  -> normalize deviceInfo/inventory/slots/sensors
  -> version-specific components
```

- 每个适配器声明所需字段并为可选数组/对象提供默认值。
- 未返回配置、空数组、未知状态和离线是不同语义，UI 不得统一显示“正常”。
- 状态码、单位换算、阈值和正常值来自设备协议或 API 文档，并在映射层集中处理。
- 新代际优先扩展适配器与专用小组件，不在旧组件的每一列追加条件分支。
- 并行详情请求都必须 settle；loading 使用 `try/finally`，不要把立即执行表达式误放进 `Promise.all`。

## 数值与空态

```js
const hasValue = value !== null && value !== undefined && value !== ''
```

- `0`、`"0"` 可能是有效温度、压力或库存，不能用 truthy 判断。
- 单位只在值有效且换算成功时附加。
- 未知状态显示“未知/未上报”，无硬件配置显示“未配置”，离线显示“离线”。
- 库存为 0 时，过期提示是否有意义由产品规则决定，不自动等同异常。
- 无商品档案的库存记录是隐藏、占位还是异常提示，应由宿主规则显式决定。

## 库存与格口

- 库存记录以稳定商品/仓位 ID 去重，名称缺失不应导致深层属性异常。
- 格口布局允许换行和动态数量；不同设备代际可使用不同网格配置。
- 门状态、占用状态、故障状态和取餐状态不要压缩成一个布尔值。
- 缺失配置、未知状态与真实空库存都提供可区分 UI 和诊断字段。

## 日志查询

- 默认日期范围由产品配置，例如最近 7 天；不要把示例天数硬编码成通用规则。
- 开始时间不得晚于结束时间，最大跨度按接口能力执行，提示文案与校验值同步。
- 筛选改变重置页码；分页 current/size 各只触发一次请求。
- 时间戳明确时区；设备时间、服务器时间和浏览器时间不可混为一谈。
- loading 在成功、失败、取消和弹窗关闭路径收口。

## 安全轮询

优先递归 `setTimeout`：

```text
start -> generation++ -> request
                    -> 若仍 active 且 generation 相同，处理响应
                    -> 请求 settle 后安排下一次
stop  -> active=false -> generation++ -> clear timer
```

- 使用单一 timer 标识和 in-flight 标记，避免重叠请求。
- 页面失活、路由离开或销毁时停止；若 `keep-alive`，同时处理 activated/deactivated。
- 旧响应必须校验 generation/revision，不能写回或重新安排 timer。
- 不对不可见详情持续轮询；列表只请求当前视图需要的字段。
- 根据错误类型退避，避免设备离线时形成请求风暴。
