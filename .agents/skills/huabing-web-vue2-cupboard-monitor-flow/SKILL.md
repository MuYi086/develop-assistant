# 话饼 Web Vue2 柜机监控流

## 开始前

1. 读取 `references/cupboard-monitor.md`。
2. 确定设备代际、页面层级、数据来源、状态枚举和刷新方式；列表、详情与日志分别建模。
3. 列出接口可能缺失的运行配置、库存、格口和传感器字段，并定义未知、无配置与正常的不同空态。

## 实施流程

1. 列表页编排筛选、统计、视图模式与刷新；卡片/统计组件只展示并上抛操作。
2. 详情页按设备代际或 capabilities 组合机器信息、库存和格口组件，不在一个模板中散落所有版本判断。
3. Vuex 仅保存确需跨组件共享的视图偏好、标签或稳定筛选；高频运行快照按页面持有。
4. 在 API 边界通过版本适配器把可选结构归一化；空数据展示明确空态，不制造伪正常状态。
5. 警报与操作日志复用日期校验、分页和 loading 语义，筛选或页码变化只发送一次请求。
6. 自动刷新可停止、不可重入；页面失活或销毁时清理定时器，并阻止旧响应写回。

## 约束

- 状态码、单位与阈值以宿主设备协议/API 文档为准，不凭常识或来源项目历史值修改。
- 压力、温度、库存数量、格口状态和过期状态分别处理；数值 `0` 不等于缺失。
- 设备代际差异进入适配器、映射或小组件，不散落到每个模板单元格。
- 视图偏好通过 mutation/action 或页面本地副本更新，不直接修改 store state。
- 使用递归 `setTimeout`、in-flight 和 generation/revision 防止请求重叠与陈旧响应复活轮询。
- 只做只读监控验证；未经明确授权不连接、控制或下发命令到真实设备。

## 验证

1. 覆盖各设备代际、在线/离线、字段完整/缺失、库存空/0/正数。
2. 覆盖传感器、门/格口、压力、温度的正常、异常、未知和无配置显示。
3. 覆盖视图切换、筛选、统计、进入详情、返回、页面缓存和轮询清理。
4. 覆盖日志默认范围、越界日期、分页、失败和 loading。
5. 运行宿主项目定向 lint/测试并审查 diff；构建前确认无部署、推送、通知或发布副作用。
