# 话饼 Web Vue2 商品库存流

## 开始前

1. 读取 `references/shop-goods-inventory.md`。
2. 确定商品形态、页面模式、活动阶段、库存模式和适用门店范围。
3. 检查宿主项目相邻页面、API 契约、ID 字段、上传组件和枚举来源；不得照搬参考字段名。

## 实施流程

1. 画出商品基本信息、计划信息、门店范围、库存模式、关联权益和提交 DTO 的转换关系。
2. 分离列表容器、编辑表单、商品明细表、共享库存、独立库存、范围选择和关联权益组件。
3. 为创建、编辑、复制、查看和运营中调整建立显式 mode；复制使用白名单 DTO，清除来源主键和不可复用状态。
4. 标签范围、手选门店和导入范围使用独立集合；切换 mode 时清理不会提交的旧选择。
5. 运营中管理库存时，在路由、查询和保存之间传递同一业务上下文 ID；缺少时阻止请求并给出明确错误。
6. 批量导入使用宿主上传封装和 FormData 契约；错误结果必须定位到行与字段。

## 关键约束

- 共享库存与独立库存的互斥不只依赖 UI 隐藏；提交前按当前 mode 校验和重建 payload。
- 时间区间、截单/履约时间、分类与活动状态按不同页面模式分别控制可编辑性。
- 弹窗关闭时重置表单、选中行、分页和校验，避免范围或库存串到下一次操作。
- `0`、`false` 是合法值；只按接口契约移除 `null`、`undefined` 或空白字符串。
- 提交按钮防重复；失败保留可修正输入，成功后以服务端结果刷新。
- 只抽取当前变更真正复用的转换函数或组件，不顺手重构整个商品域。

## 验证

1. 覆盖创建、编辑、复制、查看、运营中调整库存和重新上架。
2. 覆盖共享/独立库存、标签/手选/导入范围、空选择与模式往返切换。
3. 对比提交 DTO 与回显，确认 ID、时间、上下文、商品数量和库存字段不串模式。
4. 回归弹窗重复打开、分页/搜索、重复提交、导入部分失败和异常重试。
5. 运行宿主项目定向 lint/测试并审查 diff；构建前确认命令不触发部署、推送、通知或发布。
