# 门店商品与库存通用业务地图

## 模式矩阵

| 维度 | 典型值 | 主要风险 |
| --- | --- | --- |
| 商品形态 | 普通、组合/礼包、自提 | 字段与子商品结构不同 |
| 页面模式 | 创建、编辑、复制、查看 | ID 清理、回显、禁用项不同 |
| 业务阶段 | 草稿、运营中、结束 | 时间、范围和库存可编辑性不同 |
| 库存模式 | 共享、独立 | 汇总口径、门店数量和接口不同 |
| 门店范围 | 标签、手选、导入 | 选择集合与 payload 不可混用 |

在实现前把宿主枚举映射到矩阵；不要假设 mode、状态码或字段名与参考项目相同。

## 推荐组件边界

| 组件角色 | 责任 |
| --- | --- |
| 列表容器 | 筛选、分页、模式入口和刷新 |
| 编辑容器 | 表单编排、回显归一化和最终 DTO |
| 商品明细表 | 展示、选择、数量与行级校验 |
| 共享库存编辑器 | 一个库存池、多门店适用范围 |
| 独立库存编辑器 | 每门店独立数量、批量赋值与校验 |
| 范围选择器 | 标签、门店或导入结果的独立集合 |
| 关联权益弹窗 | 优惠券、标签或其他关联项的候选与选中结果 |

组件名与目录以宿主项目为准。子组件上抛业务结果，不直接修改页面表单或 Vuex 对象。

## 回显与复制

- API 数据进入可编辑表单前先归一化并复制。
- 创建：使用默认值，不携带任何来源 ID。
- 编辑：保留目标主键和服务端版本号，按权限控制字段。
- 复制：使用字段白名单构建新对象；查询用的 `sourceId` 不进入新增 payload。
- 清除来源对象主键、状态、审计字段和关系行主键；商品、权益与范围是否保留由需求逐项决定。
- 查看：不依赖 disabled 表单代替权限，提交入口必须不存在或被业务层阻止。

## 范围与库存

```text
rangeMode=tag    -> 只提交 tagIds
rangeMode=store  -> 只提交 storeIds
rangeMode=import -> 只提交已校验的导入集合或批次 ID
```

- 切换范围模式时清理另一集合，或在最终 DTO 中严格只读取当前集合。
- 共享库存维护库存池与适用范围；独立库存维护 `{ storeId, quantity }` 列表。
- 运营中调整必须携带稳定的上下文 ID，例如计划 ID、活动 ID 或库存池 ID；名称由接口决定。
- 缺少上下文时禁止无范围查询或保存，避免误操作全部门店。
- 并发保存可使用服务端版本号或幂等键，防止旧页面覆盖新库存。

## 数据清理与上传

- 数量、价格为 `0`，布尔值为 `false` 时必须保留。
- 日期与时区由接口契约决定；不要用展示字符串反推提交值。
- FormData 不手写错误的 JSON Content-Type，让浏览器生成 multipart boundary。
- 导入先做文件类型、大小、表头和行级校验；服务端错误要映射到具体行，允许用户下载或查看失败明细。

## 回归清单

- 新增保存后可查询，编辑回显完整，复制不覆盖来源对象。
- 活动时间、截单/履约时间和商品分类满足状态限制。
- 共享/独立库存切换无残留值，汇总口径一致。
- 标签、门店、导入三种范围的数量、去重和 DTO 一致。
- 关联弹窗取消后不污染下次打开。
- 运营中入口、查询和保存使用相同上下文 ID，普通编辑不误带旧 ID。
