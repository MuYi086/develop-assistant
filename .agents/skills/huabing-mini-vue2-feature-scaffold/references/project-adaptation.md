# 项目适配决策表

| 发现项 | 判断方式 | 处理方式 |
|---|---|---|
| SFC 顺序 | 抽样相邻 `.vue` 文件 | 沿用多数顺序，不把 Vue 3 模板带入 Vue 2 |
| 页面路由 | 查 `pages.json`、生成脚本和 package scripts | 只改权威源，运行生成命令后审查差异 |
| 分包 | 查 `subPackages`、相邻 root 和平台限制 | 复用已有分包；新增 root 前评估体积 |
| 组件注册 | 查 easycom、全局注册和局部 import | 沿用现有方式，不同时引入第二套规则 |
| 请求层 | 从页面调用追到 client/interceptor | 复用稳定 client，不在页面直接封装传输 |
| API 管理 | 查常量模块、service 或直接 client 方法 | 按既有粒度新增，不强制 API barrel |
| Store | 查 Vuex 版本、自动注册、根 getter | 只在需要共享时新增模块并同步注册机制 |
| 持久化 | 查插件、reducer、白名单和 schema | 默认不持久化，确需恢复时只保留最小字段 |
| 平台差异 | 查条件编译和适配器 | 把差异收敛在最接近平台边界的位置 |
| 测试与构建 | 查 package scripts 和 CI | 使用仓库已有命令，不发明不存在的脚本 |

## 禁止假设

- 不假设页面一定放在按字母命名的分包目录。
- 不假设请求必须经过某个名为 `util` 的模块。
- 不假设 API 必须是路径常量或必须集中 barrel 导出。
- 不假设 Vuex 自动注册、根 getter、持久化白名单存在。
- 不假设微信、支付宝、H5 和 App 共享完全相同的授权与返回结构。
