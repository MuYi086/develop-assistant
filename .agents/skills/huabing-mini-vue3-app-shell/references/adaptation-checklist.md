# Vue3 uni-app 应用壳层适配清单

## 项目画像

| 维度 | 需要确认 | 常见分支 |
| --- | --- | --- |
| 工程入口 | `main.ts`、`App.vue`、应用创建函数 | `createSSRApp`、自定义 bootstrap |
| 构建链 | 依赖、脚本、配置文件 | Vite CLI、HBuilderX、Vue CLI/webpack、自研脚本 |
| 页面源 | 页面配置及其生成方式 | `src/pages.json`、根目录 `pages.json`、生成器源文件 |
| 路由 | 导航 API 与拦截层 | uni 原生、`uni.addInterceptor`、第三方 router |
| 状态 | 全局状态和持久化 | Pinia、Vuex、组合式 store、自研状态 |
| UI | 组件与 easycom | uni-ui、uView/uv-ui、uView Plus、自研组件 |
| 样式 | 单位与转换插件 | rpx、SCSS、Tailwind、PostCSS、CSS Modules |
| 平台 | 实际交付端 | 微信、支付宝、抖音、百度等小程序及 H5/App |
| 发布 | 产物与外部动作 | `dist`、复制目录、上传 CLI、CI 发布 |

## 页面变更检查

- 页面文件与注册路径大小写一致。
- `navigationStyle`、标题、下拉刷新和滚动配置符合页面结构。
- 分包 root 不重叠，独立分包和预下载策略符合平台限制。
- 页面 auth metadata 被真实路由层消费，而不是仅存在于 JSON。
- 分享、扫码、推送、订阅消息和 WebView 回跳路径同步更新。
- 原生 tabBar 与自定义 tabBar 没有同时争夺导航状态。

## 插件与条件编译检查

- 自动扫描 glob 与文件扩展名、目录、导出形式匹配。
- 浏览器全局对象仅在 H5 分支访问。
- 小程序专用 API 只在对应 MP 分支调用。
- 插件安装顺序不早于依赖的状态容器或全局对象。
- 调试工具受 mode 控制，生产构建不输出敏感数据。

## 构建验证矩阵

至少记录以下结果：

| 平台 | 开发启动 | 生产式构建 | 页面注册 | 样式 | 登录回跳 |
| --- | --- | --- | --- | --- | --- |
| 主要小程序端 | 必测 | 必测 | 必测 | 必测 | 必测 |
| 次要小程序端 | 按范围 | 按范围 | 按范围 | 按范围 | 按范围 |
| H5（若支持） | 按范围 | 按范围 | history/base | 代理与单位 | OAuth 回跳 |

不要使用“命令退出码为 0”替代运行时验证；页面缺失、样式转换错误和插件未安装经常不会让构建失败。
