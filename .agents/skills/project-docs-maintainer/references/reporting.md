# 完成报告

默认使用中文；内容应具体、简洁、可验证，命令和文件名等可执行名称保留英文。

除非用户要求其他格式，使用以下结构：

## 已创建文档

列出新增文档；没有则写“无”。

## 已更新文档

列出修改的文档；没有则写“无”。

## 主要内容变化

概述实际完成的文档改动及其对应依据。

## 验证内容

列出实际执行的检查，例如包清单或锁文件检查、命令执行、构建或测试、链接与资源检查以及最终差异检查。

## 未验证内容

列出无法验证的事实或检查及原因；没有则写“无”。

## 缺失资源

列出仍需维护者提供的资源，例如项目标志（Logo）、当前截图、在线演示地址、性能数据、兼容性说明或发布计划。不得虚构这些资源。

## 剩余文档债务

列出与当前任务有关但超出范围的问题；没有则写“无”。

仅进行评审时，应先按严重程度列出问题，再给出本报告摘要。清楚区分已完成工作、未解决冲突和建议。
