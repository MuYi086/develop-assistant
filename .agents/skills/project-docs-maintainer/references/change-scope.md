# 变更范围与代码—文档映射

进行 `sync`（同步维护），或判断某项代码变更是否需要更新文档时，读取本文件。

## 通常需要更新文档的变更

用户可见行为、安装或启动命令、环境变量、配置字段、API（应用程序接口）契约、CLI（命令行界面）命令、路由、部署、支持平台、架构、插件或扩展点、数据格式、权限、迁移、兼容性、第三方依赖、功能状态、限制、弃用信息和安全设置发生变化时，通常需要更新相关文档。

## 通常不需要更新公开文档的变更

不改变行为的内部重构、仅格式化、仅测试、内部注释、行为无关的锁文件更新、变量改名、内部目录整理、临时调试代码和构建产物，通常不需要更新公开文档。应在完成报告中简要记录这一判断。

## 为每项文档声明查找依据

| 文档声明 | 应检查的依据 |
| --- | --- |
| 安装方式 | 锁文件、包清单、安装脚本 |
| 启动命令 | 脚本定义、入口文件、开发配置 |
| 环境变量 | `.env.example`、配置加载逻辑、数据结构定义 |
| API | 路由、控制器、接口定义、测试 |
| CLI 参数 | 命令注册和参数解析逻辑 |
| 页面路由 | 路由配置 |
| 配置项 | 类型、结构定义、默认值 |
| 部署方式 | Docker（容器平台）、Compose（多容器编排）、CI（持续集成）配置和部署脚本 |
| 平台支持 | 构建配置、测试环境、条件代码 |
| 功能状态 | 可达的实现路径和测试 |
| 许可证与版本 | 许可证文件、包清单、标签、发布记录 |

文档与代码冲突时，先确认可观察的实际行为，再检查未提交变更和开发中的实现；若仍无法确认事实来源，必须记录为未解决冲突，不得静默地选择更方便的一方。
