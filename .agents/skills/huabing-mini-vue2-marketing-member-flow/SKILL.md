# UniApp Vue 2 营销与会员流

## 领域隔离

- 优惠券、活动/组队、邀请、抽奖、积分商城和会员分别维护 API/Store 边界。
- 跨域只通过明确的 ID、选择结果和公共登录/定位/支付能力协作。
- 页面负责编排生命周期、分享、弹层和跳转；Store/service 负责状态归一和请求。
- 服务端枚举保持原值，展示 label/style 通过映射派生。

## 优惠与兑换

- 区分“个人资产列表”和“结算选择器”来源；结算返回时用 selection key 回显，不复用整页临时对象。
- 待使用、已使用、失效等 tab 使用独立 query state 或列表分页技能，切 tab 防止旧响应覆盖。
- 选择优惠先写 draft，确认后提交；服务端重新试算决定最终可用性和金额。
- 卡券、兑换码、实体卡等不同规则分别建 validator；扫码和手输进入同一归一/校验/提交链。
- blur 只做无提示的可提交性更新，正式提交完整校验并显示错误。

## 活动、邀请与抽奖

- 活动流程用显式状态机维护资格、报名/组队、待支付、成功、结束和退出。
- 邀请/渠道参数白名单化，登录注册只消费规范参数，不让活动页操作 token。
- 抽奖与领奖即使页面相似也使用不同 command，避免复用错误 endpoint。
- 动态码/活动码保存 uuid、渠道、过期时间和分享权限；过期与不可分享是业务状态。
- 分享对象由统一 builder 生成，并遵守服务端 `allowShare` 或等价权限。

## 积分与会员

- 积分商品使用列表分页、库存/积分早期校验、服务端确认和成功后余额刷新。
- 会员套餐、赠礼 draft/committed、协议同意、会员状态分别建模。
- 0 元开通也走正式提交和结果刷新；付费开通复用 payment-channel 技能。
- 会员价不可用范围由服务端能力维护，不在商品组件写死业务名单。

## 临时态与持久化

- 营销页面临时 tab、弹层草稿、抽奖动画和按钮锁默认不持久化。
- 只有跨启动必须恢复的活动上下文才持久化最小 ID、版本和过期时间。
- 活动结束、用户退出或切换账号时清理用户级营销缓存。
- 大列表与图片不进入移动端 storage。

## 验证

- 优惠：各 tab、结算回显、不可用变化、扫码/手输、重复兑换。
- 活动：未登录、无资格、报名/组队、待支付、退出、倒计时结束。
- 抽奖/邀请：有效、过期、不可分享、接口失败和重复提交。
- 积分/会员：分页、库存或积分不足、无/有赠礼、协议、0 元和付费开通。
