# 当前版本订单同步机制流程

---

## 图一：POS → Cloud 推送同步（场景一/二）

```mermaid
sequenceDiagram
    participant User as 用户/POS操作
    participant POS as POS本地服务
    participant DB as SQLite
    participant Publisher as OrderSyncPublisher
    participant Timer as 定时任务(20s)
    participant HTTP as HTTP Client
    participant Cloud as 云端 PosOrderSyncServiceImpl

    Note over User,Cloud: === 场景一：离线支付建单 → 推送 ===
    User->>POS: 创建订单(离线支付)
    POS->>DB: 保存 OrderModel<br/>localVersion=1, syncStatus=LOCAL_PENDING(0)
    POS->>Publisher: publisher(orderId)
    Publisher-->>POS: @Async → orderSyncCloud(orderId)
    POS->>DB: SELECT order + goods + sku + item<br/>+ packageCookingOrder
    POS->>HTTP: POST /pos/orderSync/local
    HTTP->>Cloud: OrderSyncCloudDTO(完整聚合根)
    
    alt 云端不存在
        Cloud->>Cloud: ACTION_CREATE<br/>save订单 + occupyOrderInventory()
        Note over Cloud: 库存扣减失败→log.warn 不回滚
    else 云端存在+无变化
        Cloud->>Cloud: ACTION_DUPLICATE<br/>仅刷新syncTime
    else 云端存在+有变化
        Cloud->>Cloud: ACTION_OVERWRITE<br/>preserveCloudAuthoritativeFields()
        alt isCloudOrderPaid()=true (在线已支付)
            Cloud->>Cloud: 保护20个支付/退款字段<br/>(payType,amountPaid,refundStatus...)
        else isCloudOrderPaid()=false
            Cloud->>Cloud: POS为权威端,全量覆盖
        end
        Cloud->>Cloud: delete旧聚合根→save新数据
    end
    
    Cloud-->>HTTP: ACK { acknowledgedVersionMap }
    HTTP-->>POS: ACK版本校验
    alt ackVer == localPushVersion
        POS->>DB: syncStatus=SYNCED(1)<br/>syncedLocalVersion=localVersion
    else ackVer不匹配
        POS->>DB: 保留LOCAL_PENDING,等待重试
    end

    Note over User,Cloud: === 场景二：订单变更触发同步 ===
    User->>POS: 修改订单(备注/取消/退款/出餐完成)
    POS->>DB: localVersion++, syncStatus=LOCAL_PENDING(0)
    POS->>Publisher: publisher(orderId) ← ✅ 本次修复补齐
    Note over Publisher: 取消操作(5处)+出餐完成(2处)均已修复
    Publisher-->>POS: @Async → 重复上述推送流程

    Note over User,Cloud: === 兜底：定时任务 ===
    Timer->>DB: SELECT syncStatus != SYNCED
    Timer->>POS: orderSyncCloud() → 重复推送流程

```

## 图二：Cloud → POS 下拉同步（场景三）

```mermaid
sequenceDiagram
    participant CloudOp as 云端操作
    participant Cloud as 云端 IOrderDeviceInteractionService
    participant MQTT as MQTT Broker
    participant IOT as IOT-Agent
    participant POS as POS OrderBizApiServiceImpl
    participant DC as OrderSyncDecisionService
    participant DB as SQLite
    participant Timer as 定时任务(20s/2min)

    Note over CloudOp,Timer: === MQTT 实时通知链路 ===
    CloudOp->>Cloud: orderSync(orderId)
    Cloud->>Cloud: 校验: orderVer>=2, 机器取餐订单
    Cloud->>Cloud: buildOrderSyncDTO()
    Cloud->>MQTT: PublishMessageSync(ORDER_SYNC)
    MQTT->>IOT: CloudCode20800Handler.handle()
    
    IOT->>IOT: ctrlMsgExecutor.execute(异步)
    IOT->>POS: HTTP POST /order/biz-api/pullCloudOrder
    Note over IOT: 失败→savePullCloudOrderRetry()<br/>定时30s重试

    POS->>Cloud: HTTP GET /pos/orderSync/posOrderPull
    Cloud-->>POS: OrderSyncCloudDTO<br/>(OrderModel+Goods+Sku+Item<br/>+PayOrder+RefundOrder<br/>+PackageCookingOrder)

    POS->>POS: applyPulledCloudOrders()
    POS->>DB: SELECT本地existingOrders

    loop 逐笔cloudOrder
        POS->>POS: resolveByVersions()→derivedStatus
        POS->>DC: decidePullAction()
        
        alt 非SITE_POS订单
            DC-->>POS: APPLY_CLOUD_SNAPSHOT
        else SITE_POS + LOCAL_PENDING
            DC-->>POS: KEEP_LOCAL_PENDING
        else SITE_POS + CONFLICT(冷却期内)
            DC-->>POS: ACKNOWLEDGE_CLOUD_VERSION
        else SITE_POS + CONFLICT(冷却期外)
            DC-->>POS: MARK_CONFLICT→记录冲突
        else SITE_POS + SYNCED/CLOUD_PENDING
            DC-->>POS: APPLY_CLOUD_SNAPSHOT
            POS->>POS: mergeSitePosLocalBusinessWithCloudPayment()
            Note over POS: BeanUtils.copyProperties(local→cloud)<br/>保留POS业务字段<br/>applyCloudPaymentAuthority()覆盖支付字段<br/>mergeRefundDetails()按refundSchId合并
        end
    end

    POS->>DB: 新订单→occupySiteGoodsInventory()
    POS->>DB: 已有订单→rollbackSiteGoodsByDelta()
    POS->>Publisher: publisher(appliedOrderId)

    Note over CloudOp,Timer: === 三层兜底(应对MQTT丢失) ===
    Timer->>Timer: L1: syncOrderData() 每20s
    Timer->>POS: pullCloudOrder(本地syncStatus≠SYNCED的订单)
    
    Timer->>Timer: L2: pullRecentCloudOrdersFallback() 每2min
    Timer->>Cloud: listRecentCloudOrderIds(按updateTime窗口)
    Timer->>POS: pullCloudOrder(云端返回的orderIds)
    
    Timer->>Timer: L3: syncStatus宽窗口查询(24h)
    Timer->>DB: SELECT syncStatus∈{CLOUD_PENDING,LOCAL_PENDING,CONFLICT}
    Timer->>POS: pullCloudOrder(兜底老旧未同步订单)

```

## 图三：关联业务同步（库存/退款/烹饪工单）

```mermaid
sequenceDiagram
    participant POS as POS本地
    participant IOT as IOT-Agent
    participant Cook as 烹饪设备
    participant Cloud as 云端
    participant Admin as 后台管理

    Note over POS,Admin: === 烹饪工单下发与状态同步 ===
    POS->>POS: 创建烹饪工单<br/>sendStatus=SENDING(2)<br/>maxSendingTimeOut=now+30s
    POS->>Cloud: orderSyncCloud→推送<br/>packageCookingOrderList含SENDING
    Cloud->>Cloud: saveAll(SENDING)
    Admin->>Cloud: 查询→显示"发送中"

    Cloud->>IOT: MQTT ORDER_SYNC
    IOT->>POS: pullCloudOrder()
    IOT->>Cook: broadcastOrder()→推送到设备
    Cook-->>IOT: 发送成功
    IOT->>POS: POST /api/onEventPackageCookingOrderSendCallback<br/>{sendStatus:SEND_SUCCESS}
    
    POS->>POS: ✅ sendSuccess()<br/>sendStatus=SEND_SUCCESS(3)<br/>cookingOrderStatus=QUEUEING
    POS->>POS: ✅ markOrderPendingSync()<br/>localVersion++, syncStatus=LOCAL_PENDING
    Note over POS: ← FIX: 本次修复,触发订单重推
    POS->>Cloud: 下一次同步→推送最新sendStatus
    Cloud->>Cloud: saveAll(SEND_SUCCESS)
    Admin->>Cloud: 查询→显示"发送成功" ✅

    Note over POS,Admin: === 库存扣减/回退 ===
    rect rgb(255,240,240)
        Note over POS,Cloud: 下单→库存扣减(主优先级:订单同步)
        POS->>POS: inventoryDeduction()→本地扣减
        POS->>Cloud: orderSyncCloud(CREATE)
        Cloud->>Cloud: occupyOrderInventory()<br/>number-=qty, soldQty+=qty
        alt 扣减失败(库存不足)
            Cloud->>Cloud: log.warn→不回滚订单存储 ✅
            Note over Cloud: ⚠️ 无自动重试/对账任务
        end
    end

    rect rgb(240,255,240)
        Note over POS,Cloud: 取消/退款→库存回滚
        POS->>POS: inventoryRollback()→本地回增
        POS->>POS: TransactionTemplate.execute()<br/>事务内:本地库存回退
        POS->>Cloud: postCloudStockGoodsBatch()<br/>事务后:HTTP通知云端回增
        alt 通知失败
            Note over POS,Cloud: ⚠️ 无补偿重试/对账任务
        end
        POS->>POS: ✅ dataTransactionCommit()<br/>localVersion++, syncStatus=LOCAL_PENDING
        POS->>Publisher: ✅ publisher(orderId) ← 本次修复
        POS->>Cloud: 推送取消+退款状态
    end

    Note over POS,Admin: === 退款明细合并(按商品粒度) ===
    rect rgb(240,240,255)
        Note over POS,Cloud: POS拉取云端退款数据
        Cloud-->>POS: pullCloudOrder→refundDetail<br/>[{refundSchId:2, goodsId:B, amount:15}]
        POS->>POS: applyCloudPaymentAuthority()
        POS->>POS: ✅ mergeRefundDetails(local, cloud)<br/>按refundSchId合并:<br/>本地{A:10}+云端{B:15}<br/>= [{A:10},{B:15}]
        Note over POS: 云端同键覆盖,本地独有保留
    end

    Note over POS,Admin: === 出餐完成→状态同步 ===
    Cook-->>IOT: 出餐完成回调
    IOT->>POS: onEventMealOrderSuccessful()
    POS->>POS: applyDeliveryToOrder()<br/>更新acquiredQty等交付字段
    POS->>POS: ✅ localVersion++, syncStatus=LOCAL_PENDING
    POS->>Publisher: ✅ publisher(orderId) ← 本次修复
    POS->>Cloud: 推送最新交付状态

    Note over POS,Admin: === 定时兜底(20s) ===
    loop 每20秒
        POS->>DB: SELECT syncStatus≠SYNCED
        POS->>Cloud: 批量推送所有待同步订单
        Note over POS,Cloud: 覆盖:烹饪工单/库存回退/退款/出餐等所有变更
    end

```

### 图例说明

| 标记 | 含义 |
| --- | --- |
| ✅ | 本次会话已修复 |
| ⚠️ | 已知遗留问题（库存对账缺失） |
| 🔵 实线 | 同步调用 |
| 🔴 虚线 | 异步/事件驱动 |

| 🔵 蓝色区域 | 退款操作 |
| 🔴 红色区域 | 订单创建 |
